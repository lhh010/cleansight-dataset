## 0. 通用判定框架（所有用例共用，避免每行重复）

### 0.1 检测类指标定义
- **TP**：预测框与真值 IoU ≥ 0.5 且类别正确。
- **通过基线（与 `config.yaml:acceptance` 一致）**：该类 `recall ≥ 0.70`、`precision ≥ 0.50`。
- **严格档（P0 关键类）**：`recall ≥ 0.80`、`precision ≥ 0.70`。
- **难点降阈档（极小 / 重遮挡 / 模糊）**：`recall ≥ 0.50`、`precision ≥ 0.50`。

### 0.2 动作识别指标定义
- **段级 F1**：预测时段与真值时段的时序 IoU，F1 ≥ 0.80 为通过。
- **帧级准确率**：frame-level accuracy ≥ 0.85。
- **过渡抖动**：动作切换处允许 ≤ 15 帧（@30fps ≈ 0.5s）误判。

### 0.3 负样本 / 边界用例
- **空帧 / 离场 / 全遮挡**：不考核 recall，但要求误检 FP = 0、不得误报类别。
- **不可能组合**：该组合误检率 = 0。

### 0.4 泛化用例
- 相对同源（seen）基线，目标类 `recall` 衰减 ≤ 15 个百分点；段 F1 ≥ 0.70。

## 2. 执行方式建议

1. **回归测试集**：把 P0 用例从 `test` / `e2e_test` split 抽帧组成固定回归集，每次模型迭代复跑，跟踪每类 recall/precision 与段 F1。
2. **负样本集**：S14/S15/S19 由 pipeline 自动构造（保留空帧 + 按 5.1 矩阵 `—` 单元采样），独立考核 FP。
3. **泛化集**：S21–S23 固定走 `e2e_test` 通道，与 train/val/test 物理隔离，只评不训。
4. **自动化校验**：每条用例的通过判定映射到 `05_check.py --strict` 的指标卡口；P0 用例失败即阻断上传。

> 详细补采/标注操作、数量测算、分阶段路线图见 `DATASET_COMPLETION_PLAN.md`。
